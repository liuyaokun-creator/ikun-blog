我刚学Python自动化测试时，踩了好几个大坑。最开始是被“自动化测试”这个词唬住了，以为要写很多代码就能搞定一切。结果一上手才发现，连个按钮都点不动，页面元素都定位不到，那时候真是懵了。后来慢慢摸索，才明白测试不是写代码，而是理解业务逻辑、掌握工具链、调试问题。

## 一、环境搭建：别让依赖卡住你

我第一次用Pytest的时候，没注意Python版本和依赖包的兼容性。运行`pip install pytest`后，直接报错说找不到模块，其实是Python3.7和pytest 6.x不兼容。当时还以为是网络问题，重装了好几次。

**操作步骤：**
1. 安装Python 3.8以上
2. 创建虚拟环境：`python -m venv myenv`
3. 激活虚拟环境：`source myenv/bin/activate`（Linux/Mac）或`myenv\Scripts\activate`（Windows）
4. 安装Pytest：`pip install pytest`

```python
# test_example.py
def test_add():
    assert 1 + 1 == 2
```

**真实踩坑案例：**
我用了Python 3.6，安装pytest后一直报错，最后发现是版本不兼容。后来换回3.8就解决了。

**解决方法：**
用`pyenv`管理多个Python版本，或者用`conda`创建隔离环境。

## 二、定位元素：Selenium的常见坑

我刚开始用Selenium的时候，以为只要写好XPath就能找到元素，结果页面加载慢，元素还没出现就执行点击，直接报错。还有时候XPath写错了，根本找不到元素。

**操作步骤：**
1. 安装Selenium：`pip install selenium`
2. 下载对应浏览器的WebDriver（比如ChromeDriver）
3. 编写定位代码：

```python
from selenium import webdriver

driver = webdriver.Chrome()
driver.get("https://example.com")
# 等待元素加载
driver.implicitly_wait(10)
element = driver.find_element_by_xpath("//button[@id='submit']")
element.click()
```

**真实踩坑案例：**
我写了一个XPath，结果发现页面是动态加载的，元素一开始不存在，导致点击失败。后来加了`implicitly_wait`才解决问题。

**给新手提个醒：** 动态页面一定要加等待，否则元素找不到是常态。

## 三、测试报告：别只写assert

我第一次写完测试用例后，只看到控制台输出“OK”，完全不知道测试结果怎么分析。后来才知道，光有assert不够，还得生成可读性强的报告。

**操作步骤：**
1. 安装pytest-html：`pip install pytest-html`
2. 运行测试并生成报告：
   `pytest --html=report.html test_example.py`

**真实踩坑案例：**
我写了十几个测试用例，但跑完后只看到“passed”，根本不知道哪些用例成功了。后来加上报告插件，才看清楚每个用例的状态。

**解决方法：** 把生成报告加入你的CI流程，方便后续查看和分析。

**新手专属建议：**
每天花30分钟练习一个定位元素的案例，1周就能熟悉Selenium的基本操作。别怕犯错，多查文档，多看别人写的测试代码。
