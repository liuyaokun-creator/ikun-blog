## 二、软件测试的核心知识点

### 1. 测试类型

软件测试主要分为**功能测试、性能测试、安全测试、兼容性测试**等。其中，**功能测试**是最基础也是最常用的。比如我们开发了一个登录接口，就需要验证用户名密码是否正确、错误提示是否合理等。

```python
# 示例：使用 Python + requests 做一个简单的功能测试
import requests

def test_login():
    url = "https://api.example.com/login"
    data = {"username": "test", "password": "123456"}
    response = requests.post(url, data=data)
    assert response.status_code == 200
    assert "success" in response.json()["message"]
```

### 2. 测试用例设计

好的测试用例应该覆盖正常流程、边界值、异常情况。比如登录时，输入空用户名、密码错误、账号不存在等场景都要测试。

```python
# 示例：测试登录失败的几种情况
def test_login_failure():
    # 空用户名
    data = {"username": "", "password": "123456"}
    response = requests.post(url, data=data)
    assert response.status_code == 400
    assert "用户名不能为空" in response.json()["error"]

    # 密码错误
    data = {"username": "test", "password": "wrong"}
    response = requests.post(url, data=data)
    assert response.status_code == 401
    assert "密码错误" in response.json()["error"]
```

### 3. 自动化测试工具

常见的工具有 **Selenium、Postman、JUnit、Pytest** 等。我刚开始用 Postman 做接口测试，后来转向 Pytest + requests 实现更灵活的自动化测试。

## 三、入门踩坑与解决方法

### 1. 脚本运行不稳定

一开始写的测试脚本经常因为网络延迟或服务器响应慢而失败，后来加了重试机制和超时设置。

```python
# 加入重试逻辑
import time

def retry_request(func, max_retries=3, delay=2):
    for i in range(max_retries):
        try:
            return func()
        except Exception as e:
            print(f"尝试 {i+1} 失败，等待 {delay} 秒后重试...")
            time.sleep(delay)
    raise Exception("请求失败")
```

### 2. 测试环境不一致

测试脚本在本地运行没问题，但部署后出错，是因为测试环境和生产环境配置不一致。后来统一使用 Docker 容器来搭建测试环境，问题少了很多。
